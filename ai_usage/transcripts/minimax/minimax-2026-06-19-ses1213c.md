## Asistan

Build modundayım. Kullanıcı "oee min max ayarlıyamıyorum" diyor — yani slider'lar artık çalışmıyor. 

Sebebi açık: yeni Slider component'inde `onChange={() => undefined}` yaptım çünkü onCommit kullanıyoruz. Bu yanlış — onChange hiç çağrılmayınca slider'ın value prop'u güncellenmediği için slider görsel olarak hareket etmiyor. Kullanıcı min max ayarlayamıyor çünkü slider DOM value'su sabit (filter'dan gelen).

Aslında input range doğrudan value ile kontrol ediliyor — sürüklerken browser value'yu günceller, ama React'in value prop'u ile eşleşmediği için React "controlled input warning" verir ve value'yu geri eşler, yani slider görsel olarak geri teper.

Düzeltme:
1. `OeeSlider` component'inde local state ile value'yu tut (zaten tutuyorum, oeeMin/oeeMax)
2. Slider component'ine value yerine `defaultValue` + uncontrolled kullan, ya da
3. `onChange` ile local state güncellesin (görsel feedback için), `onCommit` ile store güncellesin

Mevcut FilterPanel'de:
```tsx
const [oeeMin, setOeeMin] = useState<number>(filter.oeeRange.min ?? 0);
const [oeeMax, setOeeMax] = useState<number>(filter.oeeRange.max ?? 100);
```

Yani local state var. Ama Slider'a `value={oeeMin}` geçiyorum ve `onChange={() => undefined}` diyorum — bu yüzden React her onChange'inde value'yu `oeeMin`'e geri setliyor (controlled input warning) ve slider sıçrıyor.

Düzeltme: `onChange={(v) => setOeeMin(v)}` ile local state güncellesin. Bu durumda:
- Sürüklerken: local state güncellenir, slider görsel olarak hareket eder (controlled)
- Bırakınca: onCommit → store güncellenir → useRecords debounced filter ile 1 istek
